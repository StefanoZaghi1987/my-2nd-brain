## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Add `adopt_md()` to `skills/inbox-fetcher/scripts/adopt_drop.py`. This function adopts a single `.md` file from the drop zone into `raw/local/<slug>/`, mirroring `adopt_pdf()` in shape and rollback pattern.

**Depends on:** TASK-0060.01 (extraction helpers must exist first)

**Algorithm:**
1. `slug_from_filename(md_path.name)` → slug. If empty slug, return `AdoptResult(ok=False, reason=...)`
2. Collision check: if `raw/local/<slug>/` exists → return `AdoptResult(ok=False, reason=f"raw/local/{slug}/ already exists - skipped")`
3. `out_dir.mkdir(parents=True, exist_ok=True)`
4. Read file to call `extract_title_from_md()` → title (fallback: `title_from_slug(slug)`)
5. Read file to call `extract_source_url_from_md()` → source_url (may be None)
6. Write `index.md` stub — omit `source_url` line entirely if None:
```yaml
---
fetch_method: local-md
title: "<title>"
fetched: <today>
source_url: <url>   # only if found
tags: []
---

Content: [[content.md]]
```
7. On `index.md` write success: `md_path.rename(out_dir / "content.md")` (atomic move)
8. On any failure before rename: remove partial `index.md`, remove dir, raise — original stays in `raw/drop/`

**Signature:** `adopt_md(md_path: Path, local_dir: Path, dry_run: bool = False) -> AdoptResult`

Dry-run: return `AdoptResult(ok=True)` after slug derivation + collision check, without writing anything.
<!-- SECTION:DESCRIPTION:END -->
