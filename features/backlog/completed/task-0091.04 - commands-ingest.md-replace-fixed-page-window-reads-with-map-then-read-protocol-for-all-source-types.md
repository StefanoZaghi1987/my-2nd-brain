## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Replace the three source-type reading protocols in `commands/ingest.md` with the map-then-read approach, and reframe the guard.

**Three protocol replacements (see plan Task 3, steps 1–4):**

**Web articles:** Old step 2 was "Read the article body." New protocol adds Map (skim headings + opening/closing), Propose concept list (wait for user approval), Read backing sections, then Write using the structure schema.

**PDFs:** Old step 2 was "Read paper.pdf — pages 1–5 (+ last 2)." New step 2 maps the FULL heading structure throughout the entire document (no page cap) + abstract + conclusions. Then proposes concept list, reads backing sections, writes.

**Local Markdown:** Already reads full file. New steps add: Map (extract headings from already-in-context file), Propose concept list, Read backing sections (no new reads needed — file already in context).

**Guard reframe:** Replace "≤3 new pages before confirm" with "concept-list confirmation" (the map step makes the full list visible up front; user approval of the list is the gate). The ≤15 files/op invariant guard is preserved unchanged.

The exact old/new content for each replacement is in `features/plans/2026-05-30-ingest-depth-coverage.md` under "Task 3."

Commit with: `feat(ingest): map-then-read protocol — full heading scan + concept-list confirmation`
