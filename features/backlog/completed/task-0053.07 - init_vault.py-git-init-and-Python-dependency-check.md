## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Add optional git init and Python dependency check to `init_vault.py`.

**Git init:**
- If `.git` already exists in vault_dir: skip with skip() message
- Otherwise: prompt user with `input("  Initialize a git repo? [Y/n] ")`
- If yes: `subprocess.run(["git", "init", "-q"], cwd=vault_dir)`
- Before committing: check `git config user.name` and `git config user.email` with `subprocess.run([..., "--get", "user.name/email"])`. If either is empty/missing: print warn() "git user identity not configured — skipping initial commit" and skip the commit.
- If identity ok: `git add -A` then `git commit -q -m "initial vault bootstrap (v4)"`
- All git subprocess calls use `check=False` (non-fatal, same as init-vault.sh `|| true`)

**Python dep check:**
- Check for `python3` executable (Unix) or `python` (Windows) using `shutil.which()`
- If found: ok() with version from `subprocess.run([sys.executable, "--version"])`
- Collect packages from all installed SKILL.md manifests by reading `packages:` field
- For each unique package: `subprocess.run([sys.executable, "-c", f"import {import_name}"])` to test availability
- Print missing packages with install command if any are absent
