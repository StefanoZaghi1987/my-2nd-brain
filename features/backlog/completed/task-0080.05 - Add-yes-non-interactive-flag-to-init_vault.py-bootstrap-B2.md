## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**Why:** `init_vault.py` hangs in CI / automated bootstrap because two functions call `input()`: `init_git()` (line ~260, asks "Initialize a git repo?") and `install_claude_md()` (line ~380, asks "CLAUDE.md already exists. Overwrite?"). Adding `--yes` / `-y` takes safe defaults (skip git init, keep existing CLAUDE.md) so the bootstrapper can run non-interactively.

**Repo context:** `D:\my-2nd-brain` is the engine/template repo. `init_vault.py` is the cross-platform bootstrapper that creates new vaults.

**Files to modify:** `init_vault.py`
- `resolve_vault_dir()` → return `tuple[Path, bool]` (add `--yes/-y` arg; return `(vault_dir, args.yes)`)
- `init_git(vault, yes=False)` → if `yes`, skip with message "git init skipped (--yes mode)"
- `install_claude_md(vault, script_dir, yes=False)` → if `yes` and dst exists, skip with message
- `main()` → unpack `vault, yes = resolve_vault_dir()`, thread `yes=yes` through calls

**Files to create:** `tests/test_bootstrap.py`
- `test_yes_flag_completes_without_stdin` — runs `init_vault.py --yes <tmp>` with `stdin=subprocess.DEVNULL`, asserts exit 0
- `test_yes_flag_deploys_commands` — asserts `split.md`, `fetch.md`, `lint.md` exist after bootstrap
- `test_yes_flag_deploys_linkutil` — asserts `.claude/skills/shared/linkutil.py` exists after bootstrap

**All exact code in Plan Task 5** of `features/plans/2026-05-29-llm-wiki-hardening.md`.

**Dependency:** Run after TASK-0080.03 (both edit `init_vault.py` — sequential to avoid conflicts). Also relies on TASK-0080.01 (`linkutil.py` must exist for the `test_yes_flag_deploys_linkutil` test to pass).

**TDD:** Write `tests/test_bootstrap.py` → run (fails because `input()` hangs) → implement `--yes` → run (passes) → commit.
