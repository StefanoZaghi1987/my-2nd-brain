## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**Why:** `WIKILINK_RE` and `normalize_link_target` are copy-pasted verbatim in both `skills/vault-linter/scripts/lint.py` (lines 60, 168-197) and `skills/shared/find_backlinks.py` (lines 29-66) with a manual "keep in sync" comment. Any fix to link resolution must be applied in two places — a drift risk. This task extracts them into a single canonical module. While touching `lint.py`'s import block, also remove the dead `read_state` import (imported on line 37 but never called).

**Repo context:** `D:\my-2nd-brain` is the engine/template repo, NOT a deployed vault. Scripts import shared modules via `sys.path.insert(0, …/shared)` (confirmed in `lint.py:36`). The installer's `_SHARED_SCRIPTS` list (init_vault.py line 361) controls which shared files are deployed — **`linkutil.py` must be added to this list or deployed vaults won't have it.**

**Files to create:**
- `skills/shared/linkutil.py` — canonical WIKILINK_RE + normalize_link_target
- `tests/test_linkutil.py` — 9 tests for the new module

**Files to modify:**
- `skills/vault-linter/scripts/lint.py` — change import (line 37: remove `read_state`; add `from linkutil import …`); delete local WIKILINK_RE (line 60) and normalize_link_target (lines 168-197)
- `skills/shared/find_backlinks.py` — add `sys.path.insert(0, str(Path(__file__).parent))` + `from linkutil import …`; delete local copies (lines 29-66)
- `init_vault.py` — add `"linkutil.py"` to `_SHARED_SCRIPTS` list (line 361)

**All exact code and test content is in Plan Task 1** of `features/plans/2026-05-29-llm-wiki-hardening.md`. Follow TDD: write failing test → implement → verify pass → commit.
