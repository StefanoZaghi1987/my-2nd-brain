## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**Why:** `skills/shared/review_scope.py` is a fully-built, fully-tested script that determines which wiki pages have changed since the last `/review` run. However, `commands/review.md` describes this scoping logic only in English prose — the script is never invoked. If the prose and the script diverge, the LLM will silently scope reviews incorrectly. This task makes the `/review` command actually call `review_scope.py` for the default scope, following the same pattern that `/fetch` uses for `fetch_inbox.py` and `/lint` uses for `lint.py`.

**Repo context:** `D:\my-2nd-brain` is the engine/template repo. `commands/review.md` is the protocol file the LLM reads when a user runs `/review`. After `init_vault.py` deploys the vault, this file lives at `.claude/commands/review.md`. The script is deployed at `.claude/skills/shared/review_scope.py`.

**`review_scope.py` interface:**
```
python3 .claude/skills/shared/review_scope.py <vault_root>
# Exit 0: pages found (one path per line to stdout)
# Exit 1: nothing changed since last review
# Exit 2: error
```

**File to modify:** `commands/review.md` — Protocol step 2.
The exact replacement text is in **Plan Task 3** of `features/plans/2026-05-29-llm-wiki-hardening.md`.

This is a documentation-only change (no Python code modified). No test needed — verify by reading the updated file and confirming it references `review_scope.py` with the correct invocation syntax.

**Independent task** — no dependency on other tasks in this phase.
