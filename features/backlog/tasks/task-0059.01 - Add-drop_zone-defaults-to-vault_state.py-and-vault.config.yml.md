## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Add `drop_zone` section to the shared config layer so all scripts that read `load_config()` get `drop_zone.path` and `drop_zone.enabled` without requiring a user-edited config file.

**Context:** `vault_state.py` uses a `_DEFAULTS` dict that is deep-merged with `vault.config.yml`. Any new config section must be added there first, or `cfg.get("drop_zone", {})` will return `{}` even when the key is absent.

**Files to modify:**
- `skills/shared/vault_state.py` — add to `_DEFAULTS` after the `"ingest"` block
- `vault.config.yml` — append `drop_zone` section for documentation
- `tests/test_vault_state.py` — add `TestDropZoneDefaults` class

**What to add in `_DEFAULTS`:**
```python
"drop_zone": {
    "path": "raw/drop",
    "enabled": True,
},
```

**What to add in `vault.config.yml`:**
```yaml
drop_zone:
  path: "raw/drop"
  enabled: true
```
