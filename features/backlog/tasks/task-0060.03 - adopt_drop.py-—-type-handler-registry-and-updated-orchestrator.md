<!-- SECTION:DESCRIPTION:BEGIN -->
Refactor `process_drop_zone()` in `skills/inbox-fetcher/scripts/adopt_drop.py` to use a type-handler registry instead of the current two-bucket (pdf / non-pdf) split.

**Depends on:** TASK-0060.02 (adopt_md must exist)

**Registry to add (module-level constant):**
```python
HANDLERS: dict[str, Callable[[Path, Path, bool], AdoptResult]] = {
    ".pdf": adopt_pdf,
    ".md":  adopt_md,
}
```

**Replace the existing bucket split with:**
```python
supported   = [f for f in all_files if f.suffix.lower() in HANDLERS]
unsupported = [f for f in all_files if f.suffix.lower() not in HANDLERS]

for f in unsupported:
    print(f"  [!] ignored (unsupported type): {f.name}")
```

Then iterate `supported`, dispatching via `HANDLERS[f.suffix.lower()]`.

**Updated output summary** — report counts per type:
```
Found 2 PDF(s) and 1 Markdown file(s) in drop zone.
```
Single-type still works: `Found 2 PDF(s) in drop zone.`

**Dry-run summary:**
```

  would adopt: attention-is-all-you-need.pdf -> raw/local/attention-is-all-you-need/

```

The existing PDF behaviour must remain identical — this is a refactor of the dispatch layer only.
