## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Add `process_drop_zone(vault, dry_run)` and `main()` to complete `skills/inbox-fetcher/scripts/adopt_drop.py`. This is the orchestrator: reads config, scans the drop zone, calls `adopt_pdf` for each PDF, and reports results.

**Context:**
- Reads `drop_zone.path` and `drop_zone.enabled` via `load_config(vault)` (defaults added in TASK-0059.01)
- Non-PDF files in the drop zone are silently ignored (prints a warning, does not error)
- Exit code `0` = all adopted; `2` = partial (≥1 skipped due to collision); `1` = bad vault path
- `raw/local/` is created if it doesn't exist (`local_dir.mkdir(parents=True, exist_ok=True)`)
- CLI: `--vault PATH`, `--dry-run` flags

**Dependency:** TASK-0059.03 must be done first (`adopt_pdf` must exist).
