## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**Why:** When `fetch_inbox.py` fails to fetch a URL, it appends `⚠ reason` to that line in inbox.md (e.g. `- [ ] https://example.com ⚠ connection timeout`). The existing `UNCHECKED_PATTERN` regex (`^- \[ \] (https?://\S+)\s*$`) no longer matches these lines because they no longer end at the URL. The only workaround is manually deleting the `⚠ ...` text before retrying — error-prone and annoying. This task adds a `FAILED_PATTERN`, a `find_failed_entries()` function, extends `update_inbox()` to handle ⚠-marked lines, and adds a `--retry` CLI mode. A new `commands/retry.md` completes the feature.

**Repo context:** `D:\my-2nd-brain` is the engine/template repo. `fetch_inbox.py` is at `skills/inbox-fetcher/scripts/fetch_inbox.py`. After vault install, it lives at `.claude/skills/inbox-fetcher/scripts/fetch_inbox.py`.

**Files to modify:**
- `skills/inbox-fetcher/scripts/fetch_inbox.py`:
  - Add `FAILED_PATTERN = re.compile(r"^- \[ \] (https?://\S+) ⚠ .+$")` after line 81
  - Add `find_failed_entries(inbox_text: str) -> list[InboxEntry]` (same structure as `find_unchecked_entries` but uses FAILED_PATTERN; preserves sub-bullet tags/note)
  - In `update_inbox()`: after the `UNCHECKED_PATTERN.match(line)` check, add a secondary `FAILED_PATTERN.match(line)` fallback so retried ⚠-lines get updated (cleared on success, fresh reason on failure)
  - `process_vault(vault, dry_run, retry=False)`: when `retry=True`, call `find_failed_entries` instead of `find_unchecked_entries`
  - `main()`: add `--retry` argument; pass `retry=args.retry` to `process_vault`
- `init_vault.py`: add `"retry"` to `COMMANDS` list (after `"split"`)
- `tests/test_fetch_inbox.py`: add `TestFailedPattern`, `TestFindFailedEntries`, `TestUpdateInboxRetry` classes
- `tests/test_installer.py`: extend `TestCommands` with `test_retry_in_commands` and `test_retry_md_exists`

**Files to create:**
- `commands/retry.md` — protocol file: when to use, how to run (`--retry` flag), what it does/doesn't do

**All exact code, test cases, and retry.md content is in Plan Task 7** of `features/plans/2026-05-29-llm-wiki-hardening.md`.

**Dependency:** Run after TASK-0080.05 (both touch `init_vault.py` — sequential to avoid conflicts).
