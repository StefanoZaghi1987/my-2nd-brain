## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
init_vault.py::install_skills() enumerates exact script filenames to copy per skill. Adding a new .py under skills/ without updating the list silently omits it (only a warn). This is the most likely future-breakage point.

**Fix:** Add a _discover_scripts(scripts_dir) helper that returns all .py files directly under the given dir, excluding test_*.py and *_test.py. Replace the hardcoded per-skill list and _SHARED_SCRIPTS list with calls to this helper.

**Files to modify:**
- init_vault.py — add _discover_scripts() helper before install_commands(); rewrite install_skills() to auto-discover per-skill scripts/ and skills/shared/ 
- tests/test_installer.py — add TestAutoDiscover class with 3 test cases

**New helper (insert before install_commands ~line 326):**
```python
def _discover_scripts(scripts_dir: Path) -> list[Path]:
    if not scripts_dir.is_dir():
        return []
    return sorted(
        f for f in scripts_dir.iterdir()
        if f.is_file()
        and f.suffix == ".py"
        and not f.name.startswith("test_")
        and not f.name.endswith("_test.py")
    )
```

**New install_skills body:** iterates ["inbox-fetcher","vault-linter","view-builder"], calls _discover_scripts(src_dir/"scripts") per skill, then calls _discover_scripts(shared_src) for shared/. Full replacement code in plan Task 2.

**Test class TestAutoDiscover:** 3 methods — test_auto_discovers_new_script, test_original_scripts_still_installed, test_test_files_excluded. Full test code in plan Task 2.

**Verify:** pytest tests/test_installer.py passes (7 total: 4 original + 3 new).
