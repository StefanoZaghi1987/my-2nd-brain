## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Add two pure helper functions to `skills/inbox-fetcher/scripts/adopt_drop.py` that extract metadata from a Markdown file's content. These are the building blocks for `adopt_md()` (TASK-0060.02).

**Functions to add:**

`extract_title_from_md(path: Path) -> str | None`
Cascade:
1. Parse YAML frontmatter block (`---…---`) and return the `title:` value if present
2. Scan lines for the first `^# (.+)` heading and return the match
3. Return `None` (caller falls back to `title_from_slug()`)

`extract_source_url_from_md(path: Path) -> str | None`
Check frontmatter for keys in order: `source_url`, `url`, `link`, `source`. Return first non-empty string value found, or `None`.

**Constraints:**
- Standard library only — no new dependencies
- Both functions must handle files with no frontmatter, empty frontmatter, and malformed frontmatter without raising
- A minimal YAML frontmatter parser (regex-based, same approach as `lint.py`) is sufficient — no `pyyaml`

**Context:** `adopt_drop.py` already has `slug_from_filename()`, `title_from_slug()`, `AdoptResult`, and `adopt_pdf()`. Read the file before editing to follow existing patterns exactly.
