## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Add the `AdoptResult` dataclass and `adopt_pdf` function to `skills/inbox-fetcher/scripts/adopt_drop.py`. This is the core file-operation function: given a PDF path and a `local_dir`, it moves the PDF into `raw/local/<slug>/paper.pdf` and writes a stub `index.md`.

**Context:** `adopt_pdf` must:
- Derive slug via `slug_from_filename` (defined in previous task)
- Skip (return failed AdoptResult) if `local_dir / slug` already exists
- In dry_run mode: return successful AdoptResult without touching the filesystem
- Write `index.md` with `fetch_method: local-pdf`, title from `title_from_slug`, today's date, `tags: []`, and `PDF: [[paper.pdf]]`
- **No `source_url` field** — local PDFs have no origin URL

**Dependency:** TASK-0059.02 must be done first (slug helpers must exist).

**stub index.md shape:**
```yaml
---
fetch_method: local-pdf
title: "Attention Is All You Need"
fetched: 2026-05-28
tags: []
---

PDF: [[paper.pdf]]
```
